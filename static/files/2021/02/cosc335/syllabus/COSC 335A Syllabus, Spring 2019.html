Catalog description and goals

A study of the design and coding of web applications. Topics include client-side scripting, website organization, responsive web design, server-side scripting, content management systems, web application frameworks, and security. Prerequisite: COSC 330 with a minimum grade of C or COSC 350 with a minimum grade of C.

Goals

By the end of the course, you should be able to:

  1. Use HTML5 to structure content and CSS3 to style and lay out web pages and understand the distinction between structure and presentation.
  2. Understand the concept of responsive web design and be able to create responsive web pages using a CSS framework such as Bootstrap.
  3. Use JavaScript to create dynamic web pages and to validate form data.
  4. Use Ajax to update page content.
  5. Use cookies to maintain state across HTTP requests.
  6. Understand the general capabilities of a content management system (CMS) to manage a website.
  7. Use server-side scripting in to implement content-driven websites that process form inputs and pull data from a database.
  8. Understand how to use a web application framework for rapid development of web applications.

Weekly Quizzes

There will be twelve 5-point quizzes, one given each Thursday, except for Thursday of the first week of classes. A weekly quiz encourages you to keep up with the work and helps both you and me assess how well you understand the topics we are covering. Being able to rework a quiz by the next class meeting allows you to make sure that you will know or understand by the following Tuesday what you might not know or understand on a Thursday. Take advantage of being able to rework a quiz even if you think you did well on it.

Quizzes will be conducted as follows.

  1. A quiz will start toward the end of class on Thursday. All papers must be turned in when time is called at 14:15. (If you have an accommodation that calls for extra time, you may continue after time is called.)
  2. Each quiz will be worth 5 points. It will have five problems, each worth one point. An answer to a problem must be completely correct to be awarded a point. No partial credit will be given. You must show your work for any problem that requires a sequence of steps to answer in order to receive a point for that problem.
  3. You will be given a “rework” copy of the quiz in class. You have an option to rework the problems on the quiz and then submit your answers by the start of class on the following Tuesday. If all of the answers on your reworked quiz are correct, then you will earn back half of the points that you missed on the Thursday quiz—for example, if you got 2 on Thursday quiz and 5 on the rework quiz, then 3.5 will be recorded as the score for that quiz. Note: You may get all of the help you need on the rework, including help from me, except you may not post questions about the quiz at Piazza or other web forum.

Quizzes given on Thursdays are “paper and pencil only.” Cell phones and calculators may not be used. If a problem involves arithmetic and you do not feel confident doing it on paper, just write down the formula and I will evaluate it.

Coding projects

Coding projects give you practical experience with what we are covering in class. Coding projects will comprise 35% of your final grade.

Point values for projects will vary and will reflect the level of challenge. More challenging projects will have higher point values. Your coding projects score will be computed as the percentage of points earned times 35. That is, I will add up all of the points you earned for coding projects and divide that by the number of points that you could have earned and then multiply that ratio by 35.

To get a perfect score on a project, your submission must satisfy all of the requirements set forth in the project’s description and be submitted before the deadline. Often there will be a rubric available in Moodle. If you’d like to incorporate more than the required features in your code, then please do!

Project code must be submitted in Moodle. If Moodle is unavailable when you try to submit, then post a note about unavailability at Piazza and submit your project as soon as you can after Moodle becomes available again. Submissions delayed by Moodle’s unavailability will not be considered to be late submissions. I will accept projects up to 24 hours late, but will assess a 10% late penalty unless you contact me at least 24 hours before a deadline and I give you an extension.

I will occasionally give you feedback about a project in Moodle that extends an offer to improve your score. Make sure you look at any feedback I provide.

Academic integrity

The Honor Code requires faculty, staff, and students to maintain a high standard of individual honor and integrity. Work represented as your own must be your own.

How does the Honor Code apply to this course?

Writing a program is similar to writing an essay in that you are creating an original work based on your own thinking. Two authors might have the same ideas to express in an essay, but they are unlikely to express those ideas in exactly the same way—that is, using the same words, sentence structure, or development. Similarly, two programmers might come up with the same algorithm to solve a problem, but are unlikely to write exactly the same code. Two students in this course should not produce identical code for a program. “Identical” is reflected by a comparison of program structure, not necessarily by an exact comparison of the text (names, spacing, comments etc.) of a program.

You must do your own work on in-class quizzes and coding projects. However, collaboration with others on coding projects is sometimes necessary and beneficial. For this course, collaborating with others along the guidelines below is permitted and not considered a violation of the Honor Code:

  • You may collaborate with others to develop an algorithm in pseudocode, but not in Python or C++.
  • You may not copy code from another student nor may you allow another student to copy your code. In particular, copying another person's code with minor modifications in an attempt to obfuscate copying is a breach of academic integrity. Do not even let another student read your code for the purpose of determining what should go in his code.
  • You may work with others to debug code. They may help you and you may help them, but the author of the code must maintain control of the mouse and keyboard and is responsible for understanding why any changes to the code fix the bugs. If you get significant help from others, acknowledge their help in program comments.

You will occasionally be given an opportunity to pair program on a project. When pair coding, you and one other student in the class comprise an individual with respect to The Honor Code. However, both of you must work together on the code. This means that you are both present and actively engaged when coding. Only one of you doing all of the coding will be considered an Honor Code violation.

I encourage you to work with other students on homework and to collaborate as necessary to develop and debug code. Please contact me for help. If you are struggling with any aspect of the course, contact me and we will figure out a course of action.
